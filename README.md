# AI Agent для проектирования событий аналитики

Streamlit-приложение, которое помогает аналитикам и разработчикам проектировать события трекинга на основе текстовых требований. Агент анализирует требования, сопоставляет с существующими схемами событий в репозиториях `web-rudderstack` и `mobile`, и генерирует JSON-схемы новых или изменённых событий.

## Как это работает

1. Агент читает все существующие JSON-схемы событий из подключённых репозиториев (`web-rudderstack` и `mobile`)
2. Ты вставляешь текст требований к фиче — описываешь экраны, кнопки, действия пользователя
3. Агент анализирует требования, находит подходящие существующие события и предлагает новые
4. Ты редактируешь таблицу в UI (добавляешь/удаляешь строки, правишь параметры)
5. Агент генерирует финальный JSON для каждого события — готовый к добавлению в репозиторий схем

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone git@github.com:alexandrbataev/event_schema_ai_agent.git event-schema
cd event-schema
```

### 2. Создать виртуальное окружение и установить зависимости

```bash
python -m venv .venv
source .venv/bin/activate   # macOS / Linux
# .venv\Scripts\activate    # Windows
pip install -r requirements.txt
```

### 3. Настроить `.env`

Скопируй `.env.example` в `.env`:

```bash
cp .env.example .env
```

Открой `.env` и заполни:

```ini
OPENAI_API_KEY=sk-your-key-here
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini
WEB_REPO_PATH=/Users/you/path/to/web-rudderstack
MOBILE_REPO_PATH=/Users/you/path/to/mobile
```

### 4. Запустить

```bash
streamlit run main.py
```

Приложение откроется в браузере по адресу `http://localhost:8501`.

## Что можно менять

| Что менять | Где лежит | Что делает |
|---|---|---|
| **Промпт агента** | `agent.md` | Системный промпт, который определяет, как агент анализирует требования. Меняй правила, примеры, тон общения |
| **API-ключ, модель, пути к репозиториям** | `.env` | Подключение к OpenAI, пути к папкам со схемами. `.env` в `.gitignore` — секреты не улетят в репозиторий |
| **Логика агента** | `agent_logic.py` | Класс `TrackingAgent` — отправка запросов в LLM, сборка JSON-схем, парсинг ответов |
| **Чтение схем из репозиториев** | `processor.py` | Как собираются существующие события из JSON-файлов |
| **Интерфейс (Streamlit)** | `main.py` | UI: форма ввода, таблица, кнопки, отображение результата |

### Типичные сценарии доработки

- **Поменять модель** → отредактировать `OPENAI_MODEL` в `.env`
- **Поменять правила для агента** → отредактировать `agent.md` (например, добавить новые обязательные параметры, изменить формат таблицы, добавить примеры)
- **Подключить другой репозиторий со схемами** → добавить новую переменную в `.env` (например, `BACKEND_REPO_PATH=...`) и прописать её в `processor.py:ENV_PATHS`
- **Изменить внешний вид** → править `main.py` (Streamlit-компоненты)

## Запуск на Windows

Убедись, что Python установлен и добавлен в `PATH`. Команды:

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
streamlit run main.py
```
